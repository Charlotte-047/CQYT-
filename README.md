# CQYT Thesis Formatter 🎓

**一键调整论文格式，让毕业论文少折磨你一点。**

这是一个面向毕业论文 `.docx` 的自动格式整理工具：把论文丢进去，跑一下脚本，它会尽量帮你把标题、正文、摘要、表格、图片、目录保护等格式问题整理好，再自动跑一轮检查。

本项目来自真实开发痛点：论文格式这件事，真的能把一整届毕业生折磨到怀疑人生。于是我们直接做了一个 skill，把重复、机械、烦人的格式整理流程自动化。

免费、开源、能跑、能验收。能少熬一晚是一晚。

---

## 它能干什么？

简单说：**一键调整论文格式。**

它可以帮你处理：

- 论文标题层级格式；
- 正文字体、字号、缩进、对齐、行距；
- 中文摘要 / 英文摘要；
- 表格格式；
- 图片和媒体保留；
- 页眉页脚、分页等结构检查；
- 源目录保护；
- 格式化后自动验收。

跑完以后，它会告诉你是否通过当前自动检查。通过不是玄学通过，是脚本一项项扫过的那种通过。

---

## 为什么做它？

因为论文格式真的烦。

不是不会改，是太浪费时间：

- 标题一会儿黑体三号，一会儿小三；
- 正文缩进看起来差不多，实际 Word 里完全不是一回事；
- 表格边框改到怀疑鼠标；
- 图片一动就飞；
- 目录更是 Word 祖传黑魔法。

这种事情不应该靠毕业生一段一段手工折磨自己。该自动化的就自动化，该解放的就解放。

## 快速开始

安装依赖：

```bash
pip install lxml
```

格式化论文：

```bash
python scripts/format_paper_xml_only.py "input.docx" "output.docx"
```

检查结果：

```bash
python scripts/verify_all_sections_v1.py "input.docx" "output.docx"
```

如果看到：

```text
VERIFY_ALL_FAILURE_COUNT 0
RESULT 通过
```

说明当前自动验收通过。

## 目录怎么处理？

目录是 Word 里最容易炸的地方，所以当前策略是：**保护源目录**。

如果你的源文档本来有目录，工具会尽量保留它，不乱改页码、不乱重建、不乱更新。

这个策略很重要，因为目录一旦乱动，很可能出现页码错、格式乱、空行怪、字段断之类的问题。我们踩过坑，所以现在选择稳。

## 隐私说明

我们很重视用户隐私。

这个仓库不包含任何真实论文、测试论文内容、个人信息或私有输出文件。工具只处理你本地提供的文档，不会主动上传、收集或侵犯你的内容权益。

你的论文是你的，我们只负责帮它变得更像一篇正经论文。

## 使用提醒

这个工具很厉害，但最终提交前还是建议你自己打开 Word 检查一遍。

原因很简单：不同学校、不同老师、不同 Word/WPS 版本，多少都有点自己的脾气。自动化可以帮你省掉大部分机械劳动，但最终确认权还是在你手里。

建议重点看：

- 目录；
- 摘要；
- 标题；
- 正文；
- 表格；
- 图片；
- 页码；
- 页眉页脚。

不是因为它不行，而是因为毕业论文这东西，稳一点总没坏处。
